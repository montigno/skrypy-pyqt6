Shared memory for inter-process

Skrypy provides a way to use shared memory, allowing different processes to access the same block of memory to ensure the exchange of parameters between them and avoid data redundancies. The use of shared memory between processes is done both within the same machine and between client-cluster.

How to use

This requires only 2 blocks: SharedMemory_create and Shared_read
They are found in the library Environment then shared_memory


After running the 2 diagrams, the results are displayed on the console
